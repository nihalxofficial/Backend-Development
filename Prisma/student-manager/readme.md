# Express + TypeScript + Prisma 7 + Neon Setup Guide

> A clean, working setup guide for the **latest Prisma 7** with Neon PostgreSQL.

---

## Why Each Piece Exists

| Tool | Why You Need It |
|---|---|
| **Express** | Web server to handle HTTP requests |
| **TypeScript** | Type safety so you catch errors before runtime |
| **Prisma 7** | Type-safe database ORM — talk to your DB with TypeScript |
| **@prisma/adapter-pg** | Prisma 7 now **requires** a driver adapter to connect to PostgreSQL |
| **Neon** | Free cloud PostgreSQL database — no local DB setup needed |
| **dotenv** | Loads your `.env` secrets into `process.env` |
| **ts-node + nodemon** | Run TypeScript directly and auto-restart on file changes |

---

## Project Structure

```
my-app/
├── src/
│   ├── index.ts              # Express server
│   ├── prisma.ts             # Prisma client instance
│   └── generated/
│       └── prisma/           # Auto-generated by `npx prisma generate`
├── prisma/
│   └── schema.prisma         # Your DB models
├── prisma.config.ts          # Prisma 7 config (DB URL goes here for CLI)
├── .env                      # Secret env variables (never commit this)
├── .env.example              # Safe template to commit to Git
├── .gitignore
├── tsconfig.json
└── package.json
```

## Step-by-Step Setup

### 1. Create & Initialize Project

```bash
mkdir my-app && cd my-app
npm init -y
```

---

### 2. Install Dependencies

```bash
# Runtime
npm install express @prisma/client @prisma/adapter-pg pg dotenv

# Dev
npm install -D typescript ts-node nodemon prisma @types/express @types/node @types/pg
```

> **Prisma 7 requires `@prisma/adapter-pg` and `pg`** — without it, `new PrismaClient()` will throw an error.

---

### 3. Configure TypeScript

```bash
npx tsc --init
```

Replace `tsconfig.json` content with:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "rootDir": "./src",
    "outDir": "./dist",
    "strict": true,
    "esModuleInterop": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist", "prisma.config.ts"]
}
```

> `"include": ["src/**/*"]` is required — without it TypeScript picks up `prisma.config.ts` from the root and throws a `rootDir` error.

---

### 4. Create Neon Database

1. Go to **[neon.tech](https://neon.tech)** and sign up
2. Click **New Project**
3. Set:
   - **Project Name** → `my-app`
   - **Database Name** → `mydb`
4. Click **Create Project**
5. Go to **Dashboard → Connection String**
6. Copy the URL — looks like:

```
postgresql://user:password@ep-xxx.us-east-1.aws.neon.tech/mydb?sslmode=require
```

---

### 5. Create `.env`

```env
PORT=3000
DATABASE_URL="postgresql://user:password@ep-xxx.us-east-1.aws.neon.tech/mydb?sslmode=require"
```

> `sslmode=require` is mandatory for Neon connections.

---

### 6. Create `.env.example`

```env
PORT=3000
DATABASE_URL="postgresql://user:password@ep-xxx.us-east-1.aws.neon.tech/mydb?sslmode=require"
```

---

### 7. Create `.gitignore`

```
node_modules/
dist/
.env
src/generated/
```

> `src/generated/` should be in `.gitignore` — always regenerate it after cloning.

---

### 8. Initialize Prisma

```bash
npx prisma init
```

---

### 9. Configure `prisma.config.ts` (Root Folder)

> This is how Prisma 7 CLI reads your DB URL — **not** from `schema.prisma` anymore.

```ts
import 'dotenv/config';
import { defineConfig, env } from 'prisma/config';

export default defineConfig({
  schema: './prisma/schema.prisma',
  datasource: {
    url: env('DATABASE_URL'),
  },
});
```

---

### 10. Update `prisma/schema.prisma`

> In Prisma 7: provider is `prisma-client` (not `prisma-client-js`), `output` is required, and `url` is removed from schema — it lives in `prisma.config.ts`.

```prisma
generator client {
  provider = "prisma-client"
  output   = "../src/generated/prisma"
}

datasource db {
  provider = "postgresql"
}

model User {
  id    Int    @id @default(autoincrement())
  name  String
  email String @unique
}
```

---

### 11. Push Schema to Database

```bash
npx prisma db push
```

> This creates the tables in your Neon database.
> You can verify in **Neon Dashboard → Tables**.

---

### 12. Generate Prisma Client

```bash
npx prisma generate
```

> **This creates `src/generated/prisma/`** — the folder your `prisma.ts` imports from.

**If models are missing after generate, always do a clean regenerate:**

```bash
rm -rf src/generated/prisma
npx prisma generate
```

After running it, you should see:

```
src/
└── generated/
    └── prisma/
        ├── client.ts      ← import PrismaClient from here
        ├── index.d.ts
        ├── index.js
        └── ...
```

---

### 13. Create `src/prisma.ts`

> In Prisma 7, you **must** pass a driver adapter — `new PrismaClient()` alone will crash.
> Import from `./generated/prisma/client` — not `./generated/prisma` or `@prisma/client`.

```ts
import { PrismaClient } from './generated/prisma/client';
import { PrismaPg } from '@prisma/adapter-pg';
import 'dotenv/config';

const adapter = new PrismaPg({
  connectionString: process.env.DATABASE_URL!,
});

const prisma = new PrismaClient({ adapter });

export default prisma;
```

---

### 14. Create `src/index.ts`

```ts
import dotenv from 'dotenv';
dotenv.config();

import express, { Request, Response } from 'express';
import prisma from './prisma';

const app = express();
app.use(express.json());

const PORT = process.env.PORT || 3000;

// GET all users
app.get('/users', async (req: Request, res: Response) => {
  try {
    const users = await prisma.user.findMany();
    res.json(users);
  } catch (error) {
    console.error(error);
    res.status(500).json({ error: String(error) });
  }
});

// POST create user
app.post('/users', async (req: Request, res: Response) => {
  try {
    const { name, email } = req.body;
    const user = await prisma.user.create({
      data: { name, email },
    });
    res.json(user);
  } catch (error) {
    console.error(error);
    res.status(500).json({ error: String(error) });
  }
});

app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
});
```

---

### 15. Add Scripts to `package.json`

```json
"scripts": {
  "dev": "nodemon --exec ts-node src/index.ts",
  "build": "tsc",
  "start": "node dist/index.js"
}
```

---

### 16. Run the App

```bash
npm run dev
```

## Prisma 7 Key Changes (vs Prisma 6)

| | Prisma 6 | Prisma 7 |
|---|---|---|
| DB URL location | `schema.prisma` | `prisma.config.ts` |
| Driver adapter | Optional | **Required** |
| `new PrismaClient()` | Works alone | Must pass `{ adapter }` |
| Import path | `@prisma/client` | `./generated/prisma/client` |
| Generator provider | `prisma-client-js` | `prisma-client` |
| `tsconfig.json` | No `include` needed | Must add `"include": ["src/**/*"]` |

## Quick Command Reference

```bash
npx prisma init                 # Initialize Prisma
npx prisma db push              # Push schema changes to database
npx prisma generate             # Regenerate client after schema changes
rm -rf src/generated/prisma && npx prisma generate  # Clean regenerate (fixes missing models)
npx prisma studio               # Open visual DB browser
npm run dev                     # Start dev server
```

---

## When to Run Commands

| Situation | Commands |
|---|---|
| Changed `schema.prisma` | `npx prisma db push` → `npx prisma generate` |
| Models missing after generate | `rm -rf src/generated/prisma` → `npx prisma generate` |
| Changed TypeScript only | Nothing needed |
| Cloned project on new machine | `npm install` → `npx prisma generate` |
| Getting `Cannot find module './generated/prisma'` | `npx prisma generate` |
